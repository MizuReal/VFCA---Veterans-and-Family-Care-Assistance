# AGENTS.md

## Project Mission

This website represents **Veterans and Family Care Assistance**, an organization dedicated to serving veterans and their families.

Every design and implementation decision should reinforce the following values:

* Trust
* Compassion
* Dignity
* Professionalism
* Accessibility
* Clarity
* Reliability

The website should feel welcoming, respectful, and easy to use for visitors of all ages and technical abilities.

---

## Source of Truth

Before making any UI changes:

1. Read `DESIGN.md`.
2. Follow its visual language consistently.
3. Preserve all functionality.
4. Never redesign individual components in isolation.

If `DESIGN.md` conflicts with accessibility or usability, prioritize accessibility.

---

## Brand Identity

**Website Title**

Veterans and Family Care Assistance

**Tagline**

Serving those who served—and their families.

**Tone**

* Respectful
* Compassionate
* Professional
* Calm
* Encouraging
* Human

Avoid language that feels overly corporate, sales-oriented, or exaggerated.

---

## Design Principles

The interface should communicate confidence through simplicity.

Prioritize:

* Large readable typography
* Spacious layouts
* Clear visual hierarchy
* High contrast
* Consistent spacing
* Predictable navigation
* Minimal distractions

Avoid:

* Excessive gradients
* Flashy animations
* Neon colors
* Gaming aesthetics
* Crypto-inspired UI
* AI-generated visual clutter
* Decorative effects without purpose

Every component should have a reason to exist.

---

## Accessibility

Accessibility is mandatory.

Ensure:

* WCAG AA contrast compliance
* Visible keyboard focus
* Semantic HTML
* Proper heading hierarchy
* Accessible forms
* Screen reader compatibility
* Alt text for images
* Descriptive buttons and links
* Large tap targets
* Responsive layouts

Never sacrifice accessibility for appearance.

---

## Visual Style

Follow `DESIGN.md`.

In addition:

* Rounded corners should remain subtle.
* Shadows should be soft.
* Borders should be understated.
* White space should improve readability.
* Icons should be simple and consistent.
* Images should depict real veterans, military families, healthcare professionals, and community support whenever possible.

Avoid generic AI-style illustrations unless explicitly requested.

---

## Typography

Typography should prioritize readability.

Headings should be confident without being aggressive.

Body text should be easy to scan.

Maintain consistent spacing between sections.

Avoid decorative fonts.

---

## Colors

Colors should inspire confidence.

Recommended palette characteristics:

* Deep navy
* Soft blue
* White
* Neutral gray

Accent colors should be used sparingly.

Avoid overly saturated colors except where necessary for calls to action.

---

## Components

Maintain consistency across:

* Buttons
* Forms
* Cards
* Navigation
* Hero sections
* Alerts
* Tables
* Footers
* Icons

Duplicate components should be consolidated into reusable patterns.

---

## Images

Prefer imagery that represents:

* Veterans
* Military families
* Community support
* Healthcare
* Counseling
* Assistance programs
* Volunteers

Avoid stock photos that appear staged or insincere.

---

## Animations

Animations should:

* Improve understanding
* Feel subtle
* Never distract
* Respect reduced-motion preferences

Avoid unnecessary motion.

---

## Responsiveness

Every page must work on:

* Mobile
* Tablet
* Desktop
* Large desktop

Design mobile-first.

---

## Performance

Prioritize:

* Fast page loads
* Optimized images
* Lazy loading where appropriate
* Minimal layout shift
* Efficient rendering

Do not introduce unnecessary dependencies.

---

## Content

Never rewrite or remove content unless requested.

When improving layouts:

* Preserve messaging
* Preserve meaning
* Preserve accessibility

Improve presentation, not the organization's voice.

---

## AI Agent Rules

When making changes:

1. Read `DESIGN.md`.
2. Audit the existing UI.
3. Identify inconsistencies.
4. Refactor components for consistency.
5. Preserve functionality.
6. Improve accessibility.
7. Improve responsiveness.
8. Remove unnecessary visual clutter.
9. Continue refining until the interface follows one coherent design language.

Do not stop after making superficial cosmetic changes.

---

## Definition of Done

A task is complete only when:

* The UI is visually consistent.
* Accessibility standards are met.
* Responsive behavior is verified.
* No functionality has changed unintentionally.
* Components follow the design system.
* The interface communicates trust, dignity, and professionalism appropriate for veterans and their families.
